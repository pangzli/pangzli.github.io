使用github中遇到的一些问题

一般的上传过程

github的基本使用大体就是在项目根目录下用命令行来操作,具体的在下面:

  • git init
  • git add .
  • git commit -m “first commit”
  • git remote add origin http://…..
  • git push -u origin master (提交到master分支,-u是以流的形式上传,速度会变快)

有的时候会遇到一些问题

failed to push some refs to git

出现这个问题有可能是你在初始化仓库的时候初始了READEME.md,而在本地仓库没有这个文件,所以可以用下面的代码来解决

可以通过如下命令进行代码合并【注：pull=fetch+merge]

git pull --rebase origin master

fatal: remote origin already exists.

出现这个问题是github远程服务器已经存在了这个仓库,可以通过下面的方法解决

先输入$ git remote rm origin
再输入$ git remote add origin http://....
