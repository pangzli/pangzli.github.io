Ajax基本介绍

Ajax(Asynchronous JavaScript and XML),异步的JavaScript与XML.
相比于全部刷新的同步,异步的方式会大大减少从服务器请求数据的大小,典型的例子是点击加载新的评论,百度首页的智能搜索等.
当然,现在从服务器端请求的数据格式已经不单单是XML了,更多的是json数据格式,利用jQuery封装的Ajax可以更好的接受不同类型的数据格式.

Ajax中最重要的一个对象就是XMLHttpRequest，它相当于浏览器和服务器之间的一个中转站,将本来直接接触的两方分隔开,使用户操作与服务器响应分隔开,xhr并不会把所有的用户操作都提交给服务器,而是将像—些数据验证和数据处理等操作交给Ajax引擎自己来做, 只有确定需要从服务器读取新数据时再由Ajax引擎代为向服务器提交请求。

get和post两种情况的原生Ajax

原生的Ajax代码较为复杂,但是自己理解掌握了原生的Ajax代码可以帮助我们更好地使用jQuery的Ajax封装.

get方式请求数据的Ajax代码.

//1 创建XMLHttpRequest
var xhr = null;
if (XMLHttpRequest) {
    xhr = new XMLHttpRequest();
}else{
    //兼容IE6以前的老IE
    xhr = new ActiveXObject("Microsoft.XMLHTTP");
}

//2 初始化
//利用get方式发送请求,open(发送方式,url地址,是否为异步,默认为异步)
xhr.open("get","php/login.php?username=abc&password=123,true);

//3 注册XMLHttpRequest状态发生改变的事件（回调函数）
xhr.onreadystatechange = function () {
    //判断注册XMLHttpRequest的状态

    //0 XMLHttpRequest创建完对象
    //1 初始化完成 open
    //2 发送请求  send
    //3 正在接收数据
    //4 接收完成
    if (xhr.readyState == 4) {
        //判断http响应的状态码，是否成功
        if (xhr.status == 200) {
            time.innerHTML = xhr.responseText;
        }else{
            time.innerHTML = "服务器内部错误";
        }

    }
}

//4 发送请求
xhr.send(null);

向服务器发送请求的方式绝大部分是get和post两种,大部分为get,直接在url地址后面跟
?username=abc&password=123之类的参数,用&分隔.

而某些提交需要保密的内容(如帐号密码)或者需要提交的参数特别多时,此时需要使用post方式提交,这时原生的Ajax与get方式的Ajax有一定的差别.

var msg = document.getElementById("msg");

// 准备好要post的数据
var name = document.getElementById("txtName").value;
var pwd = document.getElementById("txtPwd").value;
var data = "username="+ name +"&passw="+ pwd;

//1
var xhr = null;
if (XMLHttpRequest) {
    xhr = new XMLHttpRequest();
}else{
    xhr = new ActiveXObject("Microsoft.XMLHTTP");
}
//2
xhr.open("post","php/login.php",true);

//必须在open之后。设置请求头
xhr.setRequestHeader("Content-Type","application/x-www-form-urlencoded");

//3
xhr.onreadystatechange = function () {
    if (xhr.readyState == 4) {
        if (xhr.status == 200) {
            //处理服务器返回的结果
            var result = xhr.responseText;
            //0 失败  1成功
            if (result == 1) {
                msg.innerHTML = "登录成功";
            }else{
                msg.innerHTML = "登录失败";
            }
        }else{
            msg.innerHTML = "服务器内部错误";
        }
    }
}

//4
xhr.send(data);

主体结构是一致的,都分为四步,但是还是有两三处差异.

jQuery封装的Ajax

jQuery封装的Ajax可以简单的解决这个问题,那么我们首先要知道大体的形式

$.ajax({
    type:"get", //请求方式
    cache:false,//不缓存
    url:"php/login.php", //请求的路径
    dataType:"json", //返回的数据格式,可以为xml json
    success: function (data) {
        alert(data.reason);
    },
    error: function () {
        alert("错误");
    }
});

我们所要做的大部分工作是 

success: function (data) {
    alert(data.reason);
},

在这一段中添加代码,这里是成功请求到数据后,要对数据进行处理,并在网页上呈现出来,有事需要添加很多标签,这时就可以使用一些模版,例如下面所要说的artTemplate.

jsonp

jsonp并不是另一种特殊的数据格式,而是json的一种补丁.

有时我们需要调用第三方的接口来获取数据,这时原生Ajax并不能跨域获取数据,原生的解决方法是动态的生成script标签,设置src来获取不同源服务器端的数据.服务器返回的json使用一个函数包裹，页面写一个函数，当获取到服务器返回的数据后执行该函数.

浏览器的同源策略：ajax请求的 协议、域名、端口必须一致

这时是以函数参数的形式传递数据
fn({“a”:”1”,”b”:”2”})

jQuery封装的Ajax已经封装好了jsonp格式的,当然底层的实现方法就是上面所说的动态地添加script标签.

只需要设置几个相关参数,callback参数可以不填,会动态生成一个

dataType:"jsonp",
jsonp:"",
callback:""

一个模仿百度首页的例子

实现了根据键盘录入实现搜索文本框的功能,利用了jQuery的Ajax封装和artTemplate模版,代码量大大减少.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
    <style>
        * {
            margin: 0px;
            padding: 0px;
        }
        #contents {
            margin: 100px auto;
            width: 500px;
        }
        #txt {
            width: 400px;
        }
        #datas {
            border: 1px #ccc solid;
            width: 400px;
        }
        #datas ul {
            list-style-type: none;
        }
    </style>
    <script src="js/jquery-2.2.0.js"></script>
    <script src="js/template-native-debug.js"></script>
    <script id="tpl" type="text/html">
        <div id="datas">
            <ul>
                <% for(var i=0; i< s.length; i++) { %>
                <li><%= s[i] %></li>
                <% } %>
            </ul>
        </div>
    </script>
    <script>
        $(function () {
            $("#txt").keyup(function () {
                //
                var key = $("#txt").val();
                //发送异步请求获取数据
                $.ajax({
                    type:"get",
                    url:"https://sp0.baidu.com/5a1Fazu8AA54nxGko9WTAnF6hhy/su?wd="+key,
                    dataType:"jsonp",
                    jsonp:"cb",
                    success: function (data) {
                        $("#datas").remove();

                        if (data.s.length > 0) {
                            var html = template("tpl",data);
                            $("#contents").append(html);
                        }

                    },
                    error: function () {
                        alert("出错了");
                    }
                })

        })
    </script>
</head>
<body>
    <div id="contents">
        <input type="text" name="" id="txt">
        <input type="button" value="搜索" id="btn">

</html>


常见的HTTP状态

首先需要了解1~5开头的HTTP状态码分别是什么含义：

  • 1XX 表示消息
  • 2XX 表示成功
  • 3XX 表示重定向
  • 4XX 表示请求错误
  • 5XX 表示服务器端错误

具体常见的HTTP状态码：

  • 200 OK

  • 301 Moved Permanently 重定向

    客户请求的文档在其他地方，新的URL在Location头中给出，浏览器应该自动地访问新的URL。*

  • 302 Found 临时重定向

    临时重定向，类似于301，但新的URL应该被视为临时性的替代，而不是永久性的。

  • 304 Not Modified

    客户端有缓冲的文档并发出了一个条件性的请求。服务器告诉客户，原来缓冲的文档还可以继续使用。

  • 400 Bad Request 请求出现语法错误。

  • 403 Forbidden 资源不可用。
  • 404 Not Found 无法找到指定位置的资源。

  • 500 Internal Server Error

    服务器遇到了意料不到的情况，不能完成客户的请求。

  • 501 Not Implemented

    服务器不支持实现请求所需要的功能。
